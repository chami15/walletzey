## Contexto do Projeto

**Walletzey** é uma refatoração completa do "vibewallet" — uma carteira digital pessoal para controle financeiro. O projeto novo tem:

- **Frontend**: React + TypeScript + TailwindCSS + Vite (mantido de JS/TS)
- **Backend**: Python com FastAPI (substituindo o `server/index.mjs` Node.js)
- **Banco de dados**: Supabase (PostgreSQL)
- **Agente IA**: AGNO + OpenAI `gpt-4o-mini` para análise de extratos
- **Parsing de extratos**: `docling` (PDF→Markdown) + `pandas` (CSV)
- **Gráficos de investimentos**: Grafana

O N8N foi **descartado** — toda lógica de processamento é interna ao app.

## Banco de Dados — Supabase

Tabela principal `walletzey`:

```sql
create table public.walletzey (
  id          bigint generated by default as identity not null,
  data        date null,
  debito      numeric null,
  credito     numeric null,
  valor       bigint null,
  categoria   character varying null,
  recorrente  boolean null default false,
  created_at  timestamp with time zone not null default now(),
  constraint walletzey_pkey primary key (id)
);
```

Colunas adicionais sugeridas para implementar:
- `descricao` (text) — descrição da transação
- `tipo_recorrencia` (varchar) — `'parcelado'` | `'subscription'` | `null`
- `parcelas_total` (int) — total de parcelas
- `parcela_atual` (int) — número da parcela atual
- `hash_extrato` (varchar) — hash SHA256 do extrato para deduplicação
- `fonte` (varchar) — `'extrato_pdf'` | `'extrato_csv'` | `'manual'`

## Arquitetura do Novo Sistema (Walletzey)

### Backend Python (FastAPI)

Módulos planejados:
- `api/` — routers FastAPI (transações, categorias, recorrentes, investimentos)
- `services/agent.py` — Agente AGNO + OpenAI `gpt-4o-mini` para análise de extrato
- `services/parser.py` — Parsing de PDF via `docling` e CSV via `pandas`
- `services/deduplication.py` — Detecção de extratos duplicados via hash
- `db/supabase_client.py` — Client Supabase

### Frontend React/TypeScript

Páginas planejadas:
- `/` — Dashboard: saldo em conta, gastos previstos (recorrentes), KPIs do mês + gráficos
- `/transacoes` — Lista de transações com filtro por data e categoria
- `/importar` — Upload de extrato (PDF/CSV) com processamento pelo agente
- `/recorrentes` — Cadastro de saldos recorrentes (parcelados e subscriptions)
- `/investimentos` — Consulta de investimentos via Grafana (embed)
- `/categorias` — Gerenciamento de categorias (base para o agente IA)
- `/configuracoes` — Configurações do app

### Fluxo de Importação de Extrato

```
Upload PDF/CSV → Parser (docling/pandas)
    → Agente AGNO (gpt-4o-mini)
    → Detecção de duplicatas (hash)
    → Classificação por categoria
    → Detecção de recorrências
    → Persistência no Supabase
```

## Funcionalidades Principais

1. **Dashboard KPIs** (mês atual): saldo em conta corrente, gastos previstos (recorrentes), total gasto, total ganho
2. **Importação de extrato**: PDF (via docling→markdown) ou CSV (via pandas)
3. **Agente IA**: AGNO + `gpt-4o-mini` analisa extrato, classifica categorias, detecta recorrências — usa as categorias cadastradas pelo usuário como base para não alucinar
4. **Saldos recorrentes**: cadastro de compras parceladas (N vezes) e subscriptions (dia fixo/mês)
5. **Deduplicação**: hash do extrato impede reimportação do mesmo arquivo
6. **Filtros**: por data e categoria na listagem de transações
7. **Investimentos**: aba com gráficos Grafana em tempo real
8. **Configurações**: tela com configurações relevantes ao novo fluxo
